Django's Default Template Engine

Blocks

  • Definition: Blocks are defined sections within a template that can be overridden in child templates. They allow for code reuse and modular design in templates.
  • Usage: Blocks are created using the "{% block block_name %}" and "{% endblock %}" tags. Child templates can extend parent templates and replace these blocks with specific content.

"for ... in" Loops

  • Definition: The "for ... in" loop allows iterating over iterable objects such as lists or querysets.
  • Usage: It is used within templates to dynamically generate content based on the elements of an iterable. The syntax is "{% for item in iterable %}" ... "{% endfor %}", where item represents the current element in the loop.

"if" Control Structures

  • Definition: The if control structure allows for conditional rendering of content based on specific conditions.
  • Usage: It is defined using "{% if condition %}" and "{% endif %}" tags. This structure enables the display of different content based on the truthiness of conditions, allowing for dynamic behavior within the template.

Displaying Context Variables

  • Definition: Context variables are passed from the view to the template, enabling dynamic content rendering.
  • Usage: Variables can be displayed in templates using the double curly braces syntax, "{{ variable_name }}". This will output the value of the variable in the rendered HTML.
